# 深读 · 读书辅助平台 — 开发提示词

---

## 一、项目概述

开发一个名为「深读」的个人读书辅助平台，帮助用户通过结构化的学习方法进行深度阅读，养成良好的读书习惯。

核心理念：费曼学习法 × KWL框架 × 丰田一页纸 × 遗忘曲线复习系统。

---

## 二、技术栈要求

- **前端框架**：React（函数式组件 + Hooks）
- **样式**：Tailwind CSS
- **数据存储**：浏览器 LocalStorage（数据层单独抽离为 `src/services/storage.js` 模块，所有读写操作必须通过该模块，方便后期替换为 JSON 文件或数据库）
- **AI服务**：预留 `src/services/ai.js` 模块，接口格式兼容 DeepSeek API（与 OpenAI 格式兼容），当前所有 AI 功能返回 mock 数据，后期填入 API Key 即可启用
- **图标库**：使用 Tabler Icons（`@tabler/icons-react`）
- **字体**：Google Fonts，正文使用 DM Sans，标题/书名使用 Lora（衬线字体）
- **路由**：React Router v6

## 三、整体设计规范

### 配色
- 主色（完成/确认）：`#1D9E75`（绿色），深色 `#085041`，浅底 `#E1F5EE`，边框 `#9FE1CB`
- 警示/重点：`#D85A30`（橙红），浅底 `#FAECE7`，边框 `#F5C4B3`
- 强调/重点章节：`#7F77DD`（紫色），浅底 `#EEEDFE`，边框 `#CECBF6`
- 信息/蓝色：`#378ADD`，浅底 `#E6F1FB`，边框 `#B5D4F4`
- 琥珀/警告：`#EF9F27`，浅底 `#FAEEDA`，边框 `#FAC775`
- 中性背景：使用 CSS 变量 `var(--color-background-primary/secondary/tertiary)`
- 文字：使用 CSS 变量 `var(--color-text-primary/secondary/tertiary)`
- 边框：使用 CSS 变量 `var(--color-border-tertiary/secondary)`

### 字重层级
- 页面大标题：Lora 字体，18-20px，font-weight 500
- 卡片标题/模块标题：DM Sans，13px，font-weight 500
- 正文内容：DM Sans，12-13px，font-weight 400
- 辅助说明/标签：DM Sans，10-11px，font-weight 400
- 章节序号/小数字：DM Sans，11px，font-weight 500

### 圆角
- 大卡片：12px
- 中等元素（按钮、小卡片）：8px
- 小元素（标签、徽章）：20px（胶囊形）

### 绿色使用限制
绿色**只用于**以下场景，其他地方使用中性灰色替代：
1. 已完成状态标记
2. 主操作按钮（继续阅读、提交等）
3. 进度条填充
4. Active 侧边栏项目
5. 费曼复述输入框（表示"这是用户主动输入区域"）

## 六、页面详细规格

### 6.1 书架首页 `/`

展示用户所有书籍，支持添加新书。

**书籍卡片**（网格布局，每行 3-4 本）：
- 书籍封面图（用户上传，未上传时显示绿色占位块 + 上传图标）
- 书名（Lora 字体）、作者
- 阅读进度条
- 状态标签：阅读中 / 已完成 / 未开始
- 点击进入书籍总览页

**添加新书弹窗**字段：
- 书名（必填）
- 作者
- 书籍类型（工具书 / 叙事 / 理论 / 其他）
- 阅读目的（必填，textarea，这句话贯穿整个阅读过程）
- 封面上传（支持手动上传图片；同时支持输入 ISBN，预留自动抓取封面接口，当前 mock）
- 预计阅读周期
- 章节列表录入（支持批量添加章节名称）

---

### 6.2 书籍总览页 `/book/:bookId`

**顶部书籍信息卡**：
- 左侧：书籍封面（真实封面图或占位块，点击可重新上传）
- 中间：书名（Lora 18px）、作者、类型标签组、阅读目的（带左边框的说明块）、进度条（显示百分比和当前章节）
- 右侧：「继续阅读」主按钮（绿色）、「查看总结」次级按钮

**四格数据统计**（横向排列）：
- 已完成章节 / 共 X 章
- 累计阅读时长 / 本周时长
- 待复习条目（橙红色数字）+ 「去复习」跳转链接
- 重点章节数 / 已完成数

**章节列表**（卡片内列表）：
每行显示：
- 章节序号图标（未开始：灰色数字；进行中：绿色底白字数字；已完成：绿色底勾选图标）
- 章节标题 + 副标题（小节数 / 当前进度说明）
- 右侧：星级（已完成章节显示）、状态标签、重要性标签

**重要性标签规则**（视觉区分）：
- 手动标记重点：紫色实线标签 `<星形图标> 重点章节`
- AI 建议重点：紫色虚线标签 `<机器人图标> AI 建议重点`（附带 tooltip 显示 AI 建议理由）
- 未标记：不显示标签

**阅读活跃度热力图**（近 28 天，7×4 网格）：
- 颜色深浅表示当日阅读/复习活动量，4 个深度层级
- 绿色色阶：`#E1F5EE → #9FE1CB → #5DCAA5 → #1D9E75 → #085041`
- 图例标注「少 ··· 多」

### 6.3 章节阅读页 `/book/:bookId/chapter/:chapterId`

**页面顶部**：
- 左：章节编号（小号灰色）+ 章节标题（Lora 18px 粗体）
- 右：重要性标签 + 操作按钮

**章节重要性控制**：
- 三种状态：普通章节 / 重点章节 / 忽略章节
- AI 建议时：显示建议标签 + 「确认」「降级」「忽略」三个操作按钮，以及一句 AI 建议理由
- 手动切换：下拉菜单选择状态
- 重点章节状态才显示费曼思考法模块

---

#### 6.3.1 KWL 模块（默认折叠）

**折叠状态**（始终可见）：
- 标题「阅读前 · KWL」+ 三个状态芯片：`K · X条` / `W · X个问题` / `L · 待填写 或 X条`
- 右侧「展开 / 收起」按钮

**展开状态**：
三栏网格（K / W / L）：
- **K（已知）**：列表展示已添加内容，底部「+ 添加」
- **W（想知道）**：列表展示问题，底部「+ 添加」，这些问题会作为阅读引导
- **L（学到了）**：阅读前为灰色占位，阅读后填写；章节完成后自动提示填写

---

#### 6.3.2 小节费曼复述模块

**小节切换 Tab**（水平滚动）：显示所有小节，active 状态绿色下划线

**每个小节内容**：

1. **我的费曼复述区**（优先展示，视觉权重更高）
   - 标签：「我的费曼复述」+ 小字提示「先写，再看参考」
   - 输入框：绿色背景 `#E1F5EE`，绿色边框，多行文本
   - 提交按钮：「提交复述」

2. **AI 标准概括区**（默认锁定）
   - 未提交复述时：灰色虚线框，「已提交复述后自动解锁」+ 「手动解锁」按钮
   - 点击手动解锁：先显示警告提示「提前查看会降低费曼练习效果，建议先完成自己的复述再对比」，确认后显示 AI 内容
   - 提交复述后：自动解锁，显示 AI 生成的标准概括

3. **评价与改进区**（提交复述后显示）
   三行结构：
   - ✅ 绿色圆圈图标：「正确的部分」说明
   - ⚠️ 琥珀色圆圈图标：「缺失的关键点」（缺失词语用琥珀色高亮标签显示）
   - 💡 蓝色圆圈图标：「改进建议」说明

4. **小节星级评定**（底部横条）
   - 标签「本小节掌握」+ 四颗星按钮 + 当前描述文字
   - 四档：⭐完全不会 / ⭐⭐模糊认知 / ⭐⭐⭐基本掌握 / ⭐⭐⭐⭐完全掌握
   - 选中后显示对应复习触发规则（如「7天后复习」）

---

#### 6.3.3 费曼思考法模块（仅重点章节显示）

卡片顶部：紫色标题「费曼思考法 · 深度模块」+ 「重点章节解锁」紫色标签

五个步骤（竖向排列，每步左侧紫色圆形序号）：
1. **提取核心概念**：输入框，提示「本章最重要的 1~3 个概念」
2. **第一原理追问**：输入框，提示「如果这个概念不存在，会怎样？写下你的推导…」
3. **最小可解释单元**：输入框，提示「用一句最简单的话定义这个概念…」
4. **深层知识联结**：输入框，提示「为什么它和你已知的某个知识相似？背后共同机制是什么？」
5. **一句话结论**：输入框，提示「提炼后记入问题库…」（预留问题库接口）

---

#### 6.3.4 右侧面板（摘录 / 本章进度 Tab 切换）

**摘录 Tab**：
- 顶部软限制提示：「本节 X/3 条，建议不超过 3 条」（超出 3 条变为橙色警告）
- 摘录列表：左侧绿色 2px 边框，Lora 斜体字体，底部显示页码来源
- 底部「+ 添加摘录」按钮（点击弹出输入框）

**本章进度 Tab**：
逐项显示完成状态：
- KWL 前置问题：完成 ✅ / 未开始
- 小节费曼复述：X / 总节数
- 章节费曼复述：完成 / 未开始
- 费曼思考法：X / 5（仅重点章节显示）
- 丰田一页纸：完成 / 未开始
- 思维导图：完成 / 未开始

### 6.4 章节总结页 `/book/:bookId/chapter/:chapterId/summary`

**页面顶部**：章节标题 + 「生成思维导图」按钮（绿色，含魔法棒图标）

#### 6.4.1 章节费曼复述对比

与小节费曼复述模块结构相同，但针对整章：
- 我的整章复述输入框
- AI 标准概括（同样默认锁定）
- 评价与改进三行结构
- 章节星级评定

---

#### 6.4.2 丰田一页纸总结

2×2 网格 + 底部全宽一格，共五格（自适应高度，内容撑开）：

| 背景 | 核心问题 |
|------|---------|
| 原因分析 | 对策与启示 |
| **与已有知识的联结**（全宽） |

- 「背景」格预填 AI 生成内容（可编辑）
- 其余格：点击展开输入区域，显示 「+ 点击填写…」占位
- **与已有知识的联结**（全宽格）：
  - 上方：领域标签组（可添加删除）
  - 下方：输入框「描述联结点，例如：与巴甫洛夫条件反射的共同点在于……」

---

#### 6.4.3 章节思维导图

**未生成时**：显示骨架占位结构（虚线节点和连线，展示根节点 → 三个分支 → 各两个叶子节点），底部提示「点击『生成思维导图』替换为完整版本」

**生成后**：调用 AI 服务生成节点数据，使用 React 渲染为可交互的树形思维导图（可拖拽节点、编辑节点文字）

---

#### 6.4.4 章节整体星级评定

- 标题「章节整体评定」
- 星级选择（四颗星，22px，点击选择）+ 描述文字 + 复习触发规则
- 底部说明框：「章节星级以最低小节（⭐⭐）为基准触发复习，不因整体印象良好而掩盖薄弱点」

---

#### 6.4.5 回填 KWL 的 L

展示阅读前 W 列的所有问题，逐一对应填写「实际学到了什么」，未能回答的问题标记为「待解答」

### 6.5 复习队列页 `/review`

**顶部今日状态栏**（橙红色背景卡片）：
- 左侧：时钟图标圆形 + 「今日有 X 条需要复习」+ 副标题「其中 X 条为 ⭐ 级，建议优先处理」
- 右侧：「开始复习」按钮（深橙红色）

**主体两列布局（3:2 比例）**：

**左列**：
- 今日待复习列表（卡片）
- 近期计划列表（卡片）

每条复习项显示：
- 星级
- 知识点标题
- 来源（书名 · 第 X 章）
- 复习模式图标 + 文字（跳回章节 / 卡片复习）
- 到期时间标签（今日：橙红色；即将：琥珀色；未来：灰色）

**复习模式规则**（自动判断，用户可手动切换）：
- ⭐ 第一次复习 → 跳回章节
- ⭐⭐ 第一次复习 → 跳回章节
- ⭐⭐⭐ 第一次复习 → 卡片复习
- ⭐⭐⭐⭐ 任何次数 → 卡片复习
- 第二次及以后 → 卡片复习

**右列**：卡片复习预览界面（常驻展示当前选中的复习任务）

---

#### 6.5.1 卡片复习界面

弹窗或右侧面板（深色遮罩背景），包含：
- 顶部：知识点标题 + 来源章节 + 星级 + 第 X 次复习
- 中部：复习提示问题（AI 生成，如「用自己的话解释：XXX 体现在哪些方面？」）
- 输入区：多行文本框「在此写下你的复述…」
- 底部：复习模式标签 + 「提交复述」按钮

**提交后**：
- 显示上次答案对比（上次星级此时才显示，防止锚定效应）
- AI 评价（三行结构：✅⚠️💡）
- 重新打星（四档）
- 「下一条」按钮

## 九、星级与复习系统规则

### 四档星级定义

| 星级 | 状态 | 自我测试标准 | 复习频率 |
|------|------|------------|---------|
| ⭐ | 完全不会 | 无法用任何语言描述这个概念 | 次日 → 3天 → 7天 |
| ⭐⭐ | 模糊认知 | 能说出关键词，但逻辑断裂 | 3天 → 7天 → 14天 |
| ⭐⭐⭐ | 基本掌握 | 能复述但不流畅，依赖原文结构 | 7天 → 21天 |
| ⭐⭐⭐⭐ | 完全掌握 | 流畅讲出，能举例，能联结其他知识 | 30天归档复查 |

### 两条核心原则（必须在界面层硬性保障）

**原则一：就低不就高**
章节整体星级以最低小节星级为触发基准。计算逻辑：
```javascript
const chapterEffectiveRating = Math.min(
  chapterStarRating,
  ...sectionStarRatings
)
```

**原则二：复习时隐藏上次星级**
进入复习界面时，不显示上次评定结果。用户提交复述并打星后，才展示上次星级对比。
界面实现：复习卡片初始状态隐藏历史星级字段，提交后显示。

## 十一、其他开发注意事项

1. **组件复用**：费曼复述模块（含锁定/解锁/评价/打星）需封装为独立组件 `FeynmanRecall`，小节和章节级别复用
2. **自动保存**：所有输入框内容 500ms 防抖后自动保存到 storage，不依赖手动保存按钮
3. **阅读计时**：进入章节阅读页时自动开始计时，离开页面时保存阅读时长
4. **响应式**：优先保证桌面端（1280px+）体验，移动端适配为次要目标
5. **空状态**：书架无书时显示引导添加的空状态页面
6. **加载状态**：AI 功能调用时显示 loading 状态，mock 数据可加 500ms 延迟模拟
7. **错误处理**：LocalStorage 存储失败时给用户提示
8. **问题库**：预留路由和数据结构，暂不实现具体功能，侧边栏入口显示但标注「即将上线」
